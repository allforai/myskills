# UI Forge Positioning

## One Sentence

`ui-forge` is a post-implementation UI refinement plugin for technical teams.

## Why It Exists

`frontend-design` is excellent at generating strong visual frontend output, but
it is optimized for direct interface creation. This repository needs a narrower
tool: one that works after feature implementation, respects product constraints,
and refines real code already living in a codebase.

That is the role of `ui-forge`.

## Relation to Other Plugins

- `product-design` defines design intent and UI constraints.
- `dev-forge` implements complete working features.
- `ui-forge` improves UI quality after implementation.

`ui-forge` should not absorb responsibilities from the other two.

## Intended User

Professional engineers who want technically grounded UI refinement:

- concrete file-level changes
- respect for existing architecture
- practical visual improvement
- no vague design-only output disconnected from code
