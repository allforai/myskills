# 算法一致性维度

> 仅当 project_archetype 表明项目含核心算法时由 LLM 加载。
> 适用于：CRDT、渲染引擎、压缩算法、加密实现、状态机、解析器等。

## A1 — 算法行为一致性

对比 test-vectors.json 中模型 B（多步序列）和模型 C（多角色并发）的向量：
- 目标代码对同一输入序列是否产出相同的状态转换？
- 多角色并发场景的合并结果是否一致？
- 形式化属性（交换律、结合律、幂等性等，记录在 vector 的 `property` 字段）是否成立？
- 评分 = 通过的向量数 / 总向量数 × 100
- 任何 `property` 属性失败 → 直接标记为 critical gap（算法语义断裂）

## A2 — 状态机完备性

对比 infrastructure-profile 中 `protocol_spec.state_machine` 与目标代码的状态机实现：
- 每个状态是否存在？
- 每个转换是否实现？
- 无效输入在每个状态下是否被正确拒绝？
- 评分 = 已实现转换数 / 总定义转换数 × 100

## A3 — 边界条件覆盖

LLM 读源码算法的边界条件（空输入、极值、并发冲突、溢出）→ 检查目标代码是否处理相同边界：
- 评分由 LLM 根据实际算法的边界条件数量和覆盖情况评估
