# 页面交互类型体系 v2

> 本文档是页面交互类型的**单一事实来源**（Single Source of Truth）。
> 各阶段技能引用本文档，不重复定义类型。

## 引用关系

```

  ↑ 引用

  ├── ui-design.md      → Step 4 按 interaction_type 生成视觉规格
  └── design-to-spec.md → Step 2 按 interaction_type 生成实现规格
```

## 核心设计原则

### 原则一：行为层与表现层分离

**交互类型 = 行为契约**，只描述：
- 用户意图（用户想达成什么目标）
- 操作集合（用户能执行哪些操作）
- 数据状态（实体有哪些状态和流转规则）
- 信息流向（数据如何在用户和系统间流动）

**UI = 表现决策**，放在平台实现矩阵中，不属于类型定义。

> ❌ 错误：`布局：左侧树 + 右侧编辑区`（这是 UI）
> ✅ 正确：`实体有父子层级关系，操作需携带 parentId 上下文`（这是行为）

### 原则二：类型平台无关

同一交互类型在不同平台的 UI 实现可以完全不同，行为契约相同。

### 原则三：CRUD 的 U 有两种子形态

| 子形态 | 定义 | 典型操作 |
|--------|------|---------|
| 字段编辑（Edit） | 用户自由修改实体字段值 | 修改商品名称、价格、描述 |
| 状态流转（State Transition） | 触发预定义操作，实体状态跳转，无字段编辑 | 商品上架/下架、订单发货/取消 |

## 第一轴 + 第二轴：交互类型定义与平台矩阵

### MG 管理/CRUD 类

#### MG1 只读列表

**用户意图：** 浏览、筛选、查找数据，无修改意图。

**操作集合：** 查询（分页/排序）、多维筛选、搜索、导出、点击进入详情。

**数据状态：** 无状态机，数据只读。

**信息流向：** `GET list(params)` → 展示 → 可选 `GET detail(id)` 跳转。

**平台矩阵：**

| 维度 | Web 桌面 | Mobile Web | iOS/Android | Windows | TUI |
|------|---------|------------|-------------|---------|-----|
| 进入方式 | 侧边栏菜单 | 底部 Tab / 顶部返回 | 底部 Tab / 导航栈 | 左侧导航树 | TU2 菜单选择 |
| 内容展示 | 多列数据表格 | 单列卡片列表（2-3 字段） | UITableView / RecyclerView | DataGrid 控件 | 格式化文本表格 |
| 筛选操作 | 顶部 Filter Bar 常驻 | 底部 Sheet 展开筛选 | SearchController + Sheet | 顶部筛选工具栏 | `--filter` 参数 |
| 分页方式 | 分页器（Pagination） | 上拉加载更多 | 下拉刷新 + 上拉加载 | 分页器 | 分批输出 / `--limit` |
| 导出 | 工具栏按钮 | 无 / 系统分享 | 系统分享 Sheet | 菜单栏 文件→导出 | `> output.csv` 重定向 |
| 平台特有 | 多列排序点击列头 | 触摸友好大行高 | 系统搜索集成 | 键盘快捷键导航 | ANSI 颜色高亮字段 |

#### MG2 CRUD 实体集群

**用户意图：** 对某类实体进行完整生命周期管理（创建、查看、修改、删除）。

**操作集合：** 查询列表、查看详情、新建实体、编辑字段、状态流转、删除实体（含确认）、可选批量操作。

**数据状态：** 实体存在/不存在；可含子状态（如启用/禁用）。

**信息流向：** `GET list` / `GET detail(id)` / `POST create` / `PUT update(id)` / `DELETE(id)` / 状态操作 `POST action(id)`。

MG2 实体集群由以下子屏组成，screen-map 可精确标注子类型：

| 子屏 | 代号 | 行为摘要 |
|------|------|---------|
| 实体列表 | MG2-L | 可分页查询，提供操作入口，支持批量操作 |
| 实体详情 | MG2-D | 只读展示完整字段，触发操作按钮 |
| 新建表单 | MG2-C | 空表单 → 提交 → 创建实体 |
| 字段编辑 | MG2-E | 表单回填旧值 → 修改 → 提交更新（**必须回填**） |
| 状态流转 | MG2-ST | 触发特定操作使实体状态跳转，无字段编辑表单 |

**平台矩阵（以 MG2-L 列表为基准）：**

| 维度 | Web 桌面 | Mobile Web | iOS/Android | Windows | TUI |
|------|---------|------------|-------------|---------|-----|
| 新建操作 | 工具栏按钮 | 右下角 FAB | 导航栏右侧按钮 | 工具栏按钮 | `create` 子命令 |
| 编辑承载 | 字段少→弹窗 / 字段多→独立页 | 独立页（全屏） | 独立页（push） | 弹窗 / 独立页 | 交互式表单（TU2） |
| 删除确认 | Modal 确认框 | Alert 弹层 | UIAlertController | Dialog | `y/N` 提示 |
| 状态流转触发 | 操作列下拉菜单 | 长按 / 底部操作栏 | Swipe Actions / 操作 Sheet | 右键菜单 | `action` 子命令 |
| 批量操作 | Checkbox + 工具栏 | 无 / 编辑模式简化 | iOS 编辑模式 | Checkbox + 菜单 | `--ids` 多值参数 |
| 平台特有 | 行内快速编辑 | 触摸长按预览 | Haptic 触感反馈 | 键盘快捷键 CRUD | 管道批量处理 |

#### MG3 状态机驱动

**用户意图：** 对实体执行业务操作，驱动其在预定义状态间流转（无自由字段编辑）。

**操作集合：** 仅状态操作（如发货/取消/冻结/恢复/审核通过），每个操作对应合法的状态跳转；带输入的操作（如拒绝需填理由）通过简单的确认表单完成。

**数据状态：** 实体有有限状态集（如 pending/active/frozen/cancelled），操作只能触发合法跳转。

**信息流向：** `GET list(status_filter)` → 当前状态决定可用操作 → `POST action(id)` → 服务端状态变更 → 刷新列表。

**平台矩阵：**

| 维度 | Web 桌面 | Mobile Web | iOS/Android | Windows | TUI |
|------|---------|------------|-------------|---------|-----|
| 状态展示 | 状态标签（带颜色语义） | 状态标签（带颜色） | 状态标签 / SF Symbol | 图标 + 文字 | ANSI 颜色文字 |
| 操作触发 | 操作列下拉菜单（按当前状态条件渲染） | 底部操作栏（滑出） | Swipe Actions / Action Sheet | 右键菜单 | TU2 交互式菜单 |
| 带输入操作 | Modal 小表单（如填驳回原因） | 全屏简单表单 | UIAlertController 文本输入 | Dialog 文本框 | 提示符输入 |
| 批量状态操作 | Checkbox + 工具栏批量按钮 | 无 / 简化 | 编辑模式 | Checkbox + 菜单 | `--ids` 参数 |
| 状态筛选 | 顶部状态 Tab / 下拉筛选 | 横向滚动状态 Tab | 顶部 Segment 控件 | 下拉筛选 | `--status` 参数 |
| 平台特有 | 状态变更后行内刷新（无整页刷新） | 操作后 Toast 提示 | Haptic 反馈 + 动画 | 状态栏提示 | 退出码表示结果 |

#### MG4 审批流

**用户意图：** 审查他人提交的内容或申请，做批准或驳回决策。

**操作集合：** 浏览待审队列（默认过滤待审状态）、查看详情、批准、驳回（需填驳回原因）、可选批量审批。

**数据状态：** 待审核 → 已通过 / 已驳回（不可逆）。

**信息流向：** `GET list(status=pending)` → `GET detail(id)` → `POST approve(id)` / `POST reject(id, reason)` → 通知提交方。

**平台矩阵：**

| 维度 | Web 桌面 | Mobile Web | iOS/Android | Windows | TUI |
|------|---------|------------|-------------|---------|-----|
| 队列展示 | 默认筛选待审状态的列表 | 待审列表（卡片） | 待审列表（UITableView） | 待审列表（DataGrid） | `--status=pending` 过滤列表 |
| 进入详情 | 列表行点击或「审核」链接 | 点击卡片全屏详情 | push 到详情页 | 列表行双击 | `view <id>` 子命令 |
| 批准操作 | 详情页按钮 | 底部固定按钮 | 底部操作栏 | 工具栏按钮 | `approve <id>` 命令 |
| 驳回操作 | 弹窗表单（填驳回原因） | 全屏填写驳回原因 | Alert + 文本输入 | Dialog | `reject <id> --reason "..."` |
| 批量审批 | Checkbox + 批量操作工具栏 | 无（移动端不适合批量） | 编辑模式（可多选） | Checkbox + 菜单 | `approve --ids 1,2,3` |
| 平台特有 | 键盘快捷键（A批准/R驳回） | 下拉刷新队列 | 推送通知提醒新待审 | 系统通知 | cron 脚本定时审批 |

#### MG5 主从详情

**用户意图：** 查看一个实体的完整信息，并浏览其关联的多类子实体数据。

**操作集合：** 查看主实体信息、切换子实体分类（如订单/积分/评价）、在子实体上执行各自的操作、可对主实体执行状态操作。

**数据状态：** 主实体状态 + 各子实体列表各自独立（分别分页）。

**信息流向：** `GET entity(id)` → 按需 `GET entity_orders(id, params)` / `GET entity_points(id, params)`（懒加载各子表）。

**平台矩阵：**

| 维度 | Web 桌面 | Mobile Web | iOS/Android | Windows | TUI |
|------|---------|------------|-------------|---------|-----|
| 主实体展示 | 顶部信息区（固定，不随滚动消失） | 折叠式头部（向上滚动时收起） | 固定头部 View | 顶部面板 | 首屏关键字段展示 |
| 子实体切换 | 水平 Tab 导航 | 水平滚动 Tab | UITabBarController / ViewPager | 左侧树节点 | `--section orders` 参数 |
| 子实体列表 | 各 Tab 独立分页表格 | 各 Tab 独立上拉加载 | 各 Tab 独立 UITableView | 各节点独立列表 | 分节标题 + 列表 |
| 主实体操作 | 顶部操作按钮 | 底部操作栏 / 顶部导航按钮 | 导航栏右侧按钮 / Bottom Bar | 工具栏 | `action <id>` 子命令 |
| 路由格式 | `/entities/:id` | `/entities/:id` | 导航栈参数传递 | 导航路径参数 | `view entity <id>` |
| 平台特有 | 子表内嵌操作不触发页面跳转 | 子操作全屏展示 | 子操作 push 新页 | 子操作弹窗 | 子实体交互式菜单 |

#### MG6 树形管理

**用户意图：** 管理具有父子层级关系的实体结构，维护层级 CRUD。

**操作集合：** 展开/折叠节点、新建节点（需指定父节点，即携带 parentId）、编辑节点、删除节点（需处理子节点，删除前确认）、移动节点（变更 parentId）。

**数据状态：** 树结构，节点有深度、路径、子节点数；任何写操作后树状态变更。

**信息流向：** `GET tree()` 加载完整树 → 写操作（`POST createNode({parentId})` / `PUT updateNode(id)` / `DELETE node(id)`）→ 重新加载完整树。

**平台矩阵：**

| 维度 | Web 桌面 | Mobile Web | iOS/Android | Windows | TUI |
|------|---------|------------|-------------|---------|-----|
| 树结构呈现 | 可展开折叠的树形组件 | 缩进列表（accordion） | UIOutlineView / 缩进 RecyclerView | TreeView 控件 | 字符树形（`├──` 等） |
| 选中节点 | 点击节点高亮，右侧编辑区联动 | 点击展开操作菜单 | 点击 push 详情/编辑 | 点击节点激活右侧面板 | 方向键导航，回车选中 |
| 新建子节点 | 选中父节点 → 点击新建按钮 → 表单 | 长按父节点 → 新建选项 | 操作 Sheet → 新建子节点 | 右键菜单 → 新建子节点 | `create --parent-id <id>` |
| 删除含子节点 | Modal 确认（说明子节点处理方式） | Alert 确认 | UIAlertController | Dialog | `y/N` + 子节点处理选项 |
| 节点移动 | 拖拽（可选）/ 剪切粘贴 | 不支持拖拽，通过菜单移动 | 长按拖拽（可选） | 拖拽 / 右键剪切 | `move <id> --to-parent <id>` |
| 平台特有 | 键盘展开/折叠（空格/方向键） | 触摸友好大节点高度 | 系统级拖拽支持 | 键盘快捷键 Ctrl+X/V | 完整可视化树管理器（如 `ranger`） |

#### MG7 仪表盘

**用户意图：** 获取业务全局数据视图，辅助决策，无需修改数据。

**操作集合：** 时间范围选择、数据钻取（下探到明细）、图表交互（hover/点击区域）、定时刷新（可选）。无写操作。

**数据状态：** 无状态机，纯聚合读取，数据有时效性。

**信息流向：** 并行 `GET statistics(params)` × N → 聚合展示；用户调整筛选条件 → 重新拉取。

**平台矩阵：**

| 维度 | Web 桌面 | Mobile Web | iOS/Android | Windows | TUI |
|------|---------|------------|-------------|---------|-----|
| 整体密度 | 高密度网格（多图表并排） | 垂直堆叠单列（每次展示一个图表） | 垂直滚动列表 | 可调整大小的多面板 | 字符图表（sparkline / 柱状） |
| 时间筛选 | 顶部日期范围选择器 | 底部 Sheet 时间选择 | DatePicker Sheet | 下拉日历 | `--from` / `--to` 参数 |
| 图表交互 | hover 显示 tooltip，点击钻取 | 触摸显示 tooltip | 触摸 tooltip，点击钻取 | hover tooltip | 无交互（纯输出） |
| 刷新策略 | 定时轮询 / 手动刷新按钮 | 下拉刷新 | 下拉刷新 | 自动刷新 + 状态栏 | `--watch` 定时重执行 |
| 平台特有 | 多窗口多仪表盘对比 | 关键指标卡片化（适应小屏） | 系统 Widget 支持 | 多显示器跨屏 | 可集成 CI/CD pipeline |

#### MG8 配置页

**用户意图：** 查看并修改单一实体的配置或设置，一次性提交生效。

**操作集合：** 加载当前配置值、修改字段（**必须回填已有值**）、整体保存或分区保存、可选重置为默认值。

**数据状态：** 已保存 / 修改中（本地 dirty 状态，未保存前不影响服务端）。

**信息流向：** `GET settings()` 加载 → 用户修改（本地 dirty）→ `PUT settings(values)` 保存 → 确认成功。

**平台矩阵：**

| 维度 | Web 桌面 | Mobile Web | iOS/Android | Windows | TUI |
|------|---------|------------|-------------|---------|-----|
| 表单结构 | 单页分组表单（分 Section） | 单页垂直滚动分组 | UITableView grouped style | 分组 Panel | 分区交互式表单 |
| 保存操作 | 顶部/底部「保存」按钮 | 底部固定「保存」按钮 | 导航栏右侧「储存」 | 工具栏「保存」 | `set key=value` 或 `apply` 命令 |
| 离开确认 | 有未保存修改时浏览器 beforeunload | 返回时 Alert 提示 | pop 前 UIAlertController 确认 | 关闭窗口确认 Dialog | 退出前 `y/N` 提示 |
| 重置默认 | 「恢复默认」链接/按钮 | 底部危险操作区 | 底部危险区 | 「恢复默认」菜单项 | `reset` 子命令 |
| 实时预览 | 可选（如主题配置即时预览） | 无（网络成本高） | 有（如头像裁剪） | 有（如界面设置） | 无 |
| 平台特有 | 多标签页分区配置 | 简化字段（仅必要项） | 系统设置风格 | 注册表风格嵌套 | 配置文件导入导出 |

### CT 内容消费类

#### CT1 Feed 流/信息流

**用户意图：** 在持续更新的内容流中发现感兴趣的内容，进行即时互动。

**操作集合：** 向下滚动加载更多、点赞/收藏/分享、评论、关注/取关作者、举报内容。

**数据状态：** 无限列表（游标分页），每条内容有互动计数（点赞数/评论数）状态。

**信息流向：** `GET feed(cursor)` → 渲染内容卡片 → 用户互动 `POST like(id)` / `POST follow(userId)` → 本地计数更新。

**平台矩阵：**

| 维度 | Web 桌面 | Mobile Web | iOS/Android | Windows | TUI |
|------|---------|------------|-------------|---------|-----|
| 滚动方式 | 鼠标滚轮垂直滚动 | 手指触摸垂直滚动 | 手指触摸垂直滚动 | 鼠标滚轮 | 方向键 / `j/k` 翻页 |
| 加载策略 | 滚动接近底部自动加载 | InfiniteScroll + 加载指示 | InfiniteScroll + 下拉刷新 | 滚动到底自动加载 | 按键触发加载下一批 |
| 互动操作触发 | hover 显示操作按钮 | 内容卡片内固定操作区 | 内容卡片内固定操作区 | hover 显示操作 | 选中内容后数字键操作 |
| 平台特有 | 多列瀑布流布局选项 | 触摸双击快速点赞 | Haptic 反馈 + 长按预览 | 无（少见场景） | 文本摘要 + URL，无图片 |

#### CT2 内容阅读页

**用户意图：** 完整阅读单篇文章、帖子或长内容，可互动但不编辑。

**操作集合：** 滚动阅读、点赞/收藏/分享、关注作者、查看/发表评论、举报。

### EC 电商交易类

#### EC1 商品详情页

**用户意图：** 全面了解商品信息，选择规格，决策是否购买。

**操作集合：** 浏览图片、选择规格/SKU（影响价格和库存）、设置购买数量、加入购物车、立即购买、收藏商品、查看评价列表。

**数据状态：** 规格选择状态（已选/未选各维度），选定 SKU 对应的价格/库存实时联动；加购成功状态。

**信息流向：** `GET product(id)` 加载商品基础信息 + SKU 矩阵 → 用户选择规格触发本地 SKU 匹配（价格/库存更新）→ `POST cart/add({skuId, qty})` 加购。

> **与 MG5 主从详情的区别：** EC1 的核心行为是「规格选择联动价格/库存」和「加购决策」，不是子实体管理；数据流以购买决策为终点，而非信息浏览。

**平台矩阵：**

| 维度 | Web 桌面 | Mobile Web | iOS/Android | Windows | TUI |
|------|---------|------------|-------------|---------|-----|
| 图片浏览 | 主图 + 缩略图列表，鼠标 hover 放大 | 横向滑动图片轮播 | 横向滑动（UIPageViewController） | 图片轮播组件 | N/A（图片无法在终端完整呈现，不适用） |

| 加购/购买 | 固定在视口右侧 / 底部操作区 | 底部固定操作栏 | 底部固定操作栏（系统级安全区） | 固定操作区 | N/A |

#### EC3 订单追踪时间线

**用户意图：** 了解订单当前状态和物流配送进展，无需执行任何操作。

**操作集合：** 查看时间线节点（只读）、可选点击物流节点查看详细位置信息。**无写操作。**
