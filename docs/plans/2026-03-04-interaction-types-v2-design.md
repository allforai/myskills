# 页面交互类型体系 v2 设计文档

**日期**：2026-03-04
**状态**：设计确认，待实施
**影响文件**：`product-design-skill/docs/interaction-types.md`

---

## 背景与动机

现有 v1（8 种类型 A-H）存在三个根本缺陷：

1. **平台假设隐式**：默认桌面 Web + 后台管理，未覆盖 Mobile、iOS/Android、Windows、TUI
2. **行为与 UI 混杂**：类型定义中出现"左侧树 + 右侧编辑"等 UI 描述，污染了行为层
3. **类型覆盖窄**：仅覆盖管理后台场景，缺少 Feed、IM、电商、协作、TUI 等大量主流模式

---

## 核心设计原则

### 原则一：行为层与表现层分离

**交互类型 = 行为契约**，只描述：
- 用户意图（用户想达成什么目标）
- 操作集合（用户能执行哪些操作）
- 数据状态（实体有哪些状态和流转）
- 信息流向（数据如何流动）

**UI = 表现决策**，放在平台实现矩阵中，不属于类型定义。

> 错误示例：`布局：左侧树 + 右侧编辑`（这是 UI）
> 正确示例：`实体有父子层级，操作需携带 parentId 上下文`（这是行为）

### 原则二：类型平台无关

同一交互类型在不同平台的 UI 实现可以完全不同，但行为契约相同。

### 原则三：CRUD 的 U 有两种子形态

| 子形态 | 定义 | 示例 |
|--------|------|------|
| 字段编辑（Edit） | 用户自由修改实体字段值 | 修改商品名称、价格 |
| 状态流转（State Transition） | 触发预定义操作，实体状态跳转，无字段编辑 | 商品上架/下架、订单发货 |

一个实体可以同时支持两种 U。MG2（CRUD 集群）可包含两者；MG3（状态机驱动）**仅有**状态流转，无字段编辑。

## 三轴模型

```
交互类型（行为定义层，平台无关，UI 无关）
    ×
平台实现矩阵（行为 → UI 映射，按平台差异化）
    ×
上下文预设（产品类型 × 用户属性 → 类型分布 + UI 风格倾向）
```

### MG — 管理/CRUD 类（9 种）

#### MG1 只读列表

- **用户意图**：浏览、筛选、查找数据，无修改意图
- **操作集合**：查询（含分页/排序）、筛选、搜索、导出、查看详情跳转
- **数据状态**：无状态机，数据只读
- **信息流向**：单向读取，GET list + 可选 GET detail

#### MG2 CRUD 实体集群

单个实体的完整生命周期，由以下子屏组成：

| 子屏 | 代号 | 行为描述 |
|------|------|---------|
| 实体列表 | MG2-L | 可分页查询，操作入口，含批量操作 |
| 实体详情 | MG2-D | 只读展示完整字段，可触发操作 |
| 新建表单 | MG2-C | 空表单 → 提交 → 实体创建 |
| 字段编辑 | MG2-E | 表单回填已有值 → 修改 → 提交更新 |
| 状态流转 | MG2-ST | 触发特定操作（如上架/下架），实体状态跳转，无表单 |

- **约束**：MG2-E 必须回填旧值；删除操作必须有确认步骤
- **备注**：简单实体（字段少）的 MG2-C/E 可用弹窗承载；复杂实体用独立页面

#### MG3 状态机驱动

- **用户意图**：对实体执行业务操作，驱动其状态流转
- **操作集合**：仅状态操作（发货/取消/冻结/恢复…），**无字段编辑**
- **数据状态**：实体有有限状态集，操作触发合法状态跳转
- **信息流向**：读取实体列表 → 触发操作 → 服务端状态变更 → 刷新

#### MG4 审批流

- **用户意图**：审查他人提交的内容，做批准或驳回决策
- **操作集合**：查看详情、批准、驳回（含驳回原因输入）、可选批量
- **数据状态**：待审核 → 已通过 / 已驳回
- **信息流向**：读取待审队列 → 查看详情 → 写入决策结果

#### MG5 主从详情

- **用户意图**：查看一个实体及其关联子实体的全貌
- **操作集合**：查看主实体、切换子实体分类、子实体各自的操作
- **数据状态**：主实体状态 + 各子实体列表状态独立
- **信息流向**：GET 主实体 + 按需 GET 各子实体列表（懒加载）

#### MG6 树形管理

- **用户意图**：管理具有父子层级关系的实体结构
- **操作集合**：新建节点（含 parentId）、编辑节点、删除节点（需处理子节点）、移动节点（变更 parentId）
- **数据状态**：树结构，节点有深度和路径
- **约束**：删除含子节点时需额外确认；任意写操作后需重新加载完整树

#### MG7 仪表盘

- **用户意图**：获取业务全局视图，辅助决策
- **操作集合**：时间范围筛选、图表交互（hover/drill-down），无数据写操作
- **数据状态**：无状态机，纯聚合读取
- **信息流向**：GET 统计数据（多个并行请求）→ 聚合展示

#### MG8 配置页

- **用户意图**：查看并修改单个实体的配置/设置
- **操作集合**：加载当前值、修改字段、保存（整体提交或分区保存）
- **数据状态**：已保存 / 修改中（本地 dirty 状态）
- **约束**：加载时必须回填已有值；离开时若有未保存修改需提示

### CT — 内容消费类（8 种）

#### CT1 Feed 流/信息流

- **用户意图**：浏览持续更新的内容流，发现感兴趣内容
- **操作集合**：无限加载、点赞/收藏/分享、关注/取关、举报
- **数据状态**：无限列表，内容卡片有互动计数状态
- **信息流向**：分页/游标加载 → 用户互动写回 → 本地状态更新

#### CT2 内容阅读页

- **用户意图**：阅读单篇长文/帖子的完整内容
- **操作集合**：滚动阅读、点赞/收藏、分享、进入评论区、关注作者
- **数据状态**：只读内容，有互动计数
- **信息流向**：GET 内容详情 → 评论区独立分页加载

#### CT3 个人主页/Profile

- **用户意图**：了解某用户的身份信息和内容产出
- **操作集合**：关注/取关、查看作品列表、发私信、举报
- **数据状态**：关注关系状态（已关注/未关注/互相关注）
- **信息流向**：GET 用户信息 + GET 内容列表（分 Tab 懒加载）

#### CT4 卡片探索（Swipe 模式）

- **用户意图**：快速浏览候选项，做喜欢/跳过决策
- **操作集合**：左划（跳过）、右划（喜欢/匹配）、点击查看详情、撤销上一步
- **数据状态**：候选队列，每次决策消耗一张卡片
- **信息流向**：预加载候选队列 → 决策写回 → 队列消耗后自动补充

#### CT5 媒体播放器

- **用户意图**：播放视频或音频内容
- **操作集合**：播放/暂停、进度拖拽、音量、全屏/小窗、清晰度切换、弹幕开关
- **数据状态**：播放状态机（播放中/暂停/缓冲/结束）
- **信息流向**：流媒体加载 + 进度上报 + 播放记录写回

#### CT6 相册/图库

- **用户意图**：浏览图片集合，查看大图
- **操作集合**：网格浏览、点击查看大图、左右切换、下载/分享
- **数据状态**：图片列表，当前查看索引
- **信息流向**：GET 图片列表 → 按需加载原图

#### CT7 搜索结果页

- **用户意图**：基于关键词找到符合条件的内容
- **操作集合**：输入关键词、多维筛选、排序切换、分页/加载更多、点击进入详情
- **数据状态**：搜索会话状态（关键词 + 筛选条件 + 排序 + 页码）
- **信息流向**：搜索参数 → GET 结果列表（带高亮）

#### CT8 Story/短视频流

- **用户意图**：快速刷短内容，即时娱乐
- **操作集合**：上下滑切换、点赞/评论/分享、关注、点击贴纸互动
- **数据状态**：自动播放状态，滑动切换到下一条
- **信息流向**：预加载相邻视频 → 播放完或上划 → 下一条

### EC — 电商交易类（3 种）

#### EC1 商品详情页

- **用户意图**：了解商品全貌，决策是否购买
- **操作集合**：查看图片、选择规格/SKU、设置数量、加入购物车、立即购买、收藏、查看评价
- **数据状态**：规格选择状态（影响价格/库存显示）、加购状态
- **信息流向**：GET 商品详情 → SKU 选择触发价格/库存查询 → 加购写入购物车

#### EC2 购物车/结算清单

- **用户意图**：管理待购买商品，确认数量和价格后结算
- **操作集合**：修改数量、删除商品、选择/取消选择、应用优惠券、去结算
- **数据状态**：商品列表（含选中状态）、实时价格汇总
- **信息流向**：本地状态变更 → 实时重算价格 → 结算时写入订单

#### EC3 订单追踪时间线

- **用户意图**：了解订单当前状态和物流进展
- **操作集合**：查看（无写操作），可选查看物流详情
- **数据状态**：只读时间线节点（已下单/已付款/已发货/配送中/已签收）
- **信息流向**：GET 订单状态 + GET 物流轨迹 → 展示时间线

### WK — 协作办公类（7 种）

#### WK1 对话/IM

- **用户意图**：与一个或多个对象实时通讯
- **操作集合**：发文本/图片/文件/语音消息、撤回消息、引用回复、已读标记
- **数据状态**：消息流（append-only），未读计数
- **信息流向**：实时推送（WebSocket/长连接）+ 历史消息上拉加载

#### WK2 频道/群组

- **用户意图**：在多人空间内发布内容、管理成员
- **操作集合**：发消息、@成员、固定消息、管理成员（踢出/设权限）、查看历史
- **数据状态**：频道消息流 + 成员列表 + 成员权限状态
- **信息流向**：实时推送 + 成员操作写回

#### WK3 文档编辑

- **用户意图**：创建或修改结构化文档内容
- **操作集合**：块编辑（增/删/移动块）、富文本格式化、插入媒体、版本历史查看、协同光标
- **数据状态**：文档内容状态（有 dirty/saved/conflict 状态）
- **信息流向**：OT/CRDT 实时同步 + 定期自动保存 + 版本快照

#### WK4 画布/白板

- **用户意图**：在自由坐标空间创作结构化或非结构化内容
- **操作集合**：创建节点/连线/图形、移动/缩放画布、拖拽节点、连线、批量选择
- **数据状态**：画布元素集合（各元素有坐标/尺寸/样式状态）
- **信息流向**：实时协同同步 + 操作历史（undo/redo 栈）

#### WK5 看板（Kanban）

- **用户意图**：通过列管理任务流转，可视化工作状态
- **操作集合**：创建卡片、拖拽卡片到其他列（状态变更）、编辑卡片、归档
- **数据状态**：列定义 + 各列卡片列表
- **信息流向**：拖拽触发卡片状态写回（columnId 变更）+ 实时同步

#### WK6 甘特图

- **用户意图**：在时间维度规划和跟踪任务
- **操作集合**：创建任务、拖拽调整开始/结束时间、设置依赖关系、查看进度
- **数据状态**：任务列表（含时间段、依赖关系、完成度）
- **信息流向**：时间拖拽写回任务时间字段 + 依赖关系校验

#### WK7 文件管理器

- **用户意图**：组织和操作文件/文件夹层级结构
- **操作集合**：创建文件夹、上传文件、移动（拖拽）、重命名、删除、下载、分享
- **数据状态**：当前路径 + 当前目录内容列表
- **信息流向**：路径导航 → GET 目录内容；拖拽触发移动写操作

### RT — 通讯实时类（4 种）

#### RT1 通话界面

- **用户意图**：进行语音或视频实时通话
- **操作集合**：静音/取消静音、开/关摄像头、扬声器切换、挂断、切换至小窗
- **数据状态**：通话状态机（连接中/通话中/结束）
- **信息流向**：WebRTC 媒体流 + 信令服务器

#### RT2 直播间

- **用户意图**：观看实时视频内容并与主播/其他观众互动
- **操作集合**：点赞、发弹幕/评论、送礼物、关注主播、分享
- **数据状态**：直播状态（直播中/已结束）、弹幕流（实时 append）
- **信息流向**：视频流拉取 + WebSocket 弹幕推送 + 礼物写回

#### RT3 邮件收发

- **用户意图**：接收、阅读、回复和发送邮件
- **操作集合**：收件箱浏览、阅读邮件、回复/转发、撰写新邮件、标记/归档/删除
- **数据状态**：已读/未读、标签、归档状态
- **信息流向**：IMAP 拉取 + SMTP 发送，草稿自动保存

#### RT4 通知中心

- **用户意图**：查看系统和业务通知，处理待办提醒
- **操作集合**：标记已读、批量清除、点击跳转到相关实体
- **数据状态**：已读/未读，按时间分组
- **信息流向**：GET 通知列表（分页）+ 实时推送新通知 + 已读写回

### SB — 审核提交类（1 种）

#### SB1 审核型提交

- **用户意图**：将自己创建的内容/实体提交给平台审核，等待生效
- **操作集合**：
  - 创建草稿（可保存不提交）
  - 提交审核（进入待审队列）
  - 查看审核状态（待审/审核中/通过/驳回）
  - 驳回后修改并重新提交
  - 撤回已提交（回到草稿）
- **数据状态**：`草稿 → 待审核 → 审核中 → 已通过 / 已驳回 → (修改) → 待审核`
- **信息流向**：提交触发进入 MG4 队列；驳回通知返回提交方；通过后实体生效
- **关联**：与 MG4（审批流）是同一工作流的两端（SB1 = 提交方视角，MG4 = 审核方视角）
- **典型场景**：商品上架、内容发布、广告投放、商家入驻、提现申请、KYC 认证

### SY — 引导系统类（2 种）

#### SY1 Onboarding/引导流程

- **用户意图**：了解产品核心功能，完成首次配置，授权必要权限
- **操作集合**：滑动/点击浏览步骤、填写初始信息、授权权限、跳过（可选）
- **数据状态**：引导完成状态（未开始/进行中/已完成）
- **信息流向**：单向向导，完成后写入用户 onboarding 状态

#### SY2 向导/多步表单

- **用户意图**：通过分步骤完成一个复杂的创建/配置任务
- **操作集合**：填写当前步骤、下一步/上一步、暂存草稿、最终提交
- **数据状态**：当前步骤索引、各步骤完成状态、整体草稿状态
- **信息流向**：步骤数据本地暂存 → 最终一次性提交 / 或按步保存

### TU — TUI/CLI 类（4 种）

#### TU1 命令行交互

- **用户意图**：通过文本命令执行操作，获取结构化输出
- **操作集合**：输入命令（含参数/flags）、查看输出、管道传递、Ctrl+C 中断
- **数据状态**：无 UI 状态机，状态在命令输出中体现
- **信息流向**：stdin → 命令执行 → stdout/stderr 输出

#### TU2 交互式菜单（TUI）

- **用户意图**：通过键盘导航选择操作项
- **操作集合**：方向键移动光标、回车确认、Esc 返回、Space 多选
- **数据状态**：当前光标位置、已选项集合
- **信息流向**：键盘事件驱动渲染更新，确认后触发操作

#### TU3 日志流/监控

- **用户意图**：实时查看系统运行输出，排查问题
- **操作集合**：关键词过滤、暂停/继续滚动、保存到文件
- **数据状态**：日志 append-only 流，过滤条件
- **信息流向**：stdout/stderr 实时 append → 过滤渲染

#### TU4 进度/任务流

- **用户意图**：跟踪多步骤任务的执行进展
- **操作集合**：查看（只读），可选 Ctrl+C 中断
- **数据状态**：各步骤状态（等待/进行中/完成/失败）
- **信息流向**：任务执行 → 进度事件 → 渲染更新

### 第二轴：平台实现矩阵结构

每种类型内嵌一张矩阵，描述在各平台的 UI 实现差异。矩阵维度：

| 实现维度 | 说明 |
|---------|------|
| 导航模式 | 如何进入该页面（侧边栏/底部 Tab/路由跳转） |
| 主要操作触发 | 操作如何被触发（表格操作列/底部 Sheet/右键菜单） |
| 内容展示形式 | 数据如何呈现（表格/卡片/列表/全屏） |
| 写操作承载 | 表单放在哪里（Modal/独立页/抽屉/内联） |
| 平台特有能力 | 该平台独有的能力（推送通知/手势/文件系统访问） |

覆盖平台：`Web桌面` / `Mobile Web(H5)` / `iOS+Android原生` / `Windows桌面` / `TUI`

### 第三轴：上下文预设结构

#### 产品类型预设

```
产品类型：[电商 | 社交 | IM | 社区 | 工具 | 办公 | 金融 | 内容/媒体]
→ 高频类型分布（哪些类型占比高）
→ 典型端构成（有哪些子端）
→ 推荐导航结构
```

#### 用户属性预设

```
用户属性：[C端消费者 | B端商户/企业 | 内部办公]
→ 信息密度：低/中/高
→ 错误容忍度：高（友好提示）/ 中 / 低（精确错误）
→ 操作频率：低频偶发 / 中频日常 / 高频重复
→ 学习成本接受度：低 / 中 / 高
```

## 跨端分布示例

### 操作日志（最单纯）

| 端 | 交互类型 | 备注 |
|---|---------|------|
| 运营后台 Web | MG1 只读列表 | 多维筛选，无写操作 |
| 商家后台 Web | MG1 只读列表 | 简化版，仅查自己 |
| 消费者 App | MG1 只读列表 | 登录记录/安全日志，极简 |
| TUI/监控 | TU3 日志流 | 实时 append，可 grep |

**结论**：操作日志跨端几乎永远是 MG1，唯一变量是字段密度和筛选维度。

### 商品（变化最大）

| 端 | 交互类型 | 备注 |
|---|---------|------|
| 消费者 App 列表 | CT1 Feed 流 | 算法推荐，瀑布流 |
| 消费者 App 详情 | EC1 商品详情页 | 图片轮播+规格+加购 |
| 消费者 Web 列表 | CT7 搜索结果页 | 左侧筛选+右侧网格 |
| 消费者 Web 详情 | EC1 商品详情页 | 同 App 但布局更宽 |
| 商家后台 列表 | MG2 CRUD 集群 | 含 MG2-L/C/E/ST |
| 商家后台 新建/编辑 | SY2 向导多步表单 | 字段多，分步填写 |
| 商家后台 上架下架 | MG2-ST 状态流转 | 操作触发状态跳转 |
| 商家端 提交审核 | SB1 审核型提交 | 草稿→提交→等待结果 |
| 运营后台 审核 | MG4 审批流 | SB1 的对端 |
| 运营后台 管理 | MG2 CRUD 集群 | 含批量上下架 |

**结论**：同一"商品"实体跨端涉及 7 种不同交互类型。

### 订单（最复杂）

| 端 | 交互类型 | 备注 |
|---|---------|------|
| 消费者 App 列表 | MG1 只读列表 | 我的订单，状态筛选 Tab |
| 消费者 App 详情 | MG5+MG3 主从详情+状态机 | 商品项+物流；可取消/退款 |
| 消费者 App 追踪 | EC3 订单追踪时间线 | 物流节点，只读 |
| 商家后台 列表 | MG3 状态机驱动 | 发货/取消/备注，批量操作 |
| 商家后台 退款处理 | MG4 审批流 | 查看退款申请，批准/驳回 |
| 运营后台 列表 | MG3 状态机驱动 | 更多筛选维度 |
| 运营后台 统计 | MG7 仪表盘 | GMV/转化漏斗 |
| TUI 运维脚本 | TU1 命令行交互 | 查询状态，触发补偿 |

**结论**：订单是跨端类型分裂最明显的实体，一个实体对应 8 种不同交互类型。
